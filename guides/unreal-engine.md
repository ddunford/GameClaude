## 2. World Partition & streaming
- Server streaming is **opt-in and off by default**: `wp.Runtime.EnableServerStreaming=1` makes the server stream around player sources instead of loading the whole map. Without it the server loads everything (CPU pinned in production). `[web]` [Epic KB тАФ WP Server Streaming](https://forums.unrealengine.com/t/world-partition-server-streaming/688971)
- **`wp.Runtime.EnableServerStreamingOut` stays `0`.** Streaming a cell OUT destroys its actors тАФ on an authoritative server that is data loss. `[SRC: architecture.md ┬з7]` тЪа **re-verify** (the "streaming-out = actor destruction" causal claim is web-unconfirmed). Two mechanics of leaving it `0` *are* cited: the server first activates all non-data-layer cells (`WorldPartitionStreamingPolicy.cpp:442-454`), then its loaded set is **monotonically increasing** тАФ every already-loaded cell is re-added every frame (`:506-517`), so residency converges on the **union of every cell any player has visited since start-up**. Server streaming buys *deferred* loading, not *bounded* residency: budget server memory against the **full map**, and treat a restart as the only reset. Also: the server will not deactivate a cell while any client still has it visible, so one AFK player pins it for everyone (`:522-537`). Interim lever to pin only stateful layers resident: `wp.Runtime.ServerDisallowStreamingOutDataLayers` (a comma-separated allowlist, `:46-51`). Prefer the per-map `EWorldPartitionServerStreamingMode` in World Settings (`WorldPartition.h:71-86`) over the global cvar тАФ source-controlled and immune to ini mistakes. Re-enable streaming-out only once persistence-on-unload exists. `[SRC: WorldPartitionStreamingPolicy.cpp:442-454,506-517,522-537]`
- **WP grid cell size + loading range are UI-only in practice** тАФ `UWorldPartition::RuntimeHash` is `protected` and `AWorldSettings::WorldPartition` is `VisibleAnywhere`, so MCP/Python can't set them; it's a World Settings тЦ╕ World Partition Setup action. `[MEASURED: 2026-07-19]` ┬╖ `[web]` [Epic тАФ World Partition](https://dev.epicgames.com/documentation/en-us/unreal-engine/world-partition-in-unreal-engine)
- Defaults kept consciously: **12800 uu cells / 25600 uu loading range**. Loading range is a radius тЖТ doubling it is ~4├Ч loaded set/memory. `[verify]`
- **Changing cell size later re-buckets every actor and forces a full HLOD rebuild.** `[verify]`
- **HLODs are a build step, not automatic** (Build тЦ╕ Build HLODs, or the WP HLODs commandlet) тАФ and not currently triggerable from the agent toolset. `[web]` [Epic тАФ World Partition](https://dev.epicgames.com/documentation/en-us/unreal-engine/world-partition-in-unreal-engine)
- **HLOD layer assets clone from `/Engine/Maps/Templates/HLODs/HLODLayer_Instanced`/`_Merged`; repoint the clone's `parentLayer` at the local copy** or it keeps an engine dependency. A map from the OpenWorld template silently points `default_hlod_layer` at `/Engine/OpenWorldTemplate/HLOD/HLOD0_Instancing` тАФ repoint it. `[MEASURED: 2026-07-19]`
- **`wp.Runtime.HLOD.ForceDisable` does NOT exist in 5.8** (a widespread community claim). The real control is the `wp.Runtime.HLOD` console *command* (`HLODRuntimeSubsystem.cpp:147-192`, default on) тАФ being a command it **cannot be set from ini**; use `-ExecCmds="wp.Runtime.HLOD 0"`. The server **does** load HLOD cells (`ShouldSkipDisabledHLODCell` has no `bIsServer` term, `:540-543`, while the perf-skip directly below it does). тЪа Whether HLOD proxy meshes carry server-side collision is unverified. `[SRC: HLODRuntimeSubsystem.cpp:147-192,540-543]`

### Data Layers тАФ the client/server split, and why not for districts
- **Data Layers are NOT a district mechanism** тАФ `AWorldDataLayers` is `bAlwaysRelevant = true`, so every client receives the full state of every layer regardless of distance; a layer-per-district scales the wrong way. Keep runtime layer count in the **tens**. Districts are **spatial** (WP cells + streaming sources handle them for free); the per-actor levers are `RuntimeGrid` (an `FName` on `AActor`) and `bIsSpatiallyLoaded`. `[SRC: WorldDataLayers.cpp:54-78]`
- **Query a layer's *effective* state, not its requested state** тАФ four replicated arrays exist: `Loaded`/`Active` are requested, `EffectiveLoaded`/`EffectiveActive` are resolved after the parent hierarchy. `[SRC]` (architecture.md ┬з6)
- **Data Layer load filters give client-only / server-only content, enforced in engine:** `EDataLayerLoadFilter { None, ClientOnly, ServerOnly }` (`DataLayerAsset.h:17-26`); the server is *rejected* from changing a `ClientOnly` layer (`ClientOnlyFromServer`, `WorldDataLayers.cpp:103`) and the client from a `ServerOnly` one (`WorldDataLayers.cpp:84-119`). `ClientOnly` тЖТ cosmetics, signage detail, ambient VFX, client crowds/traffic: zero server memory, physics and replication тАФ **the mechanism behind "ambient life is client-side only".** `ServerOnly` тЖТ spawn logic, AI nav, anti-cheat volumes, district-ruleset-enforcement volumes clients never receive (a "never trust the client" win). Requires `IsRuntime()` тАФ an Editor-type layer silently ignores the filter; `InitialRuntimeState` is unsupported for client/server-only layers (`DataLayerInstance.h:237`). `[SRC: DataLayerAsset.h:17-26; WorldDataLayers.cpp:84-119]`
- **A load filter is authored on the ASSET but only ever consulted through an INSTANCE.** `LoadFilter` lives on `UDataLayerAsset` (`DataLayerAsset.h:87-88`, `IsClientOnly()` at `:70`), but streaming reads it via a `UDataLayerInstance` (`UDataLayerInstanceWithAsset::IsClientOnly()` forwards to the asset, `DataLayerInstanceWithAsset.h:46`; the base returns `false`, `DataLayerInstance.h:157`). **An asset assigned to an actor with no `DataLayerInstance` in that world does nothing тАФ the actor loads everywhere.** Each world needs its own instance. `[MEASURED: 2026-07-18]` ┬╖ `[SRC: DataLayerAsset.h:70,87-88; DataLayerInstance.h:157]`
- **`ClientOnly` layers do not load on a dedicated server** тАФ verified on a live `-server` run: a `ClientOnly` marker *with an instance* was absent from `obj list`, the same asset *without an instance* present. `[MEASURED: 2026-07-18]` тЪа **A LISTEN server with server streaming still loads client-only cells** (`WorldPartitionRuntimeHash.cpp:615-620`) тАФ the filter is not a security boundary on a listen host (Epic treats the area as unfinished: `//@todo_ow` at `WorldPartitionRuntimeCell.h:357`). Harmless here only because listen servers are never used for authority work.

## 3. Replication / Iris
- **Iris status, 5.8:** the plugin metadata still reads Beta (`Iris.uplugin`: `IsBetaVersion:true`, `VersionName:0.1`, `EnabledByDefault:false`) `[SRC: Iris.uplugin]`, while the **5.8 release notes call Iris "production-ready for licensees"** `[web]` [UE 5.8 Release Notes](https://dev.epicgames.com/documentation/unreal-engine/unreal-engine-5-8-release-notes). Net: usable and promoted, but Beta still permits breaking API changes across upgrades тАФ don't claim packaged-build correctness.
- **Enabling Iris is coordinated across four places:** plugin `"Enabled":true` in `.uproject`; `bUseIris=true` in `*.Target.cs` + `SetupIrisSupport(Target)` in `.Build.cs`; cvars `net.Iris.UseIrisReplication=1` **and** `net.SubObjects.DefaultUseSubObjectReplicationList=1`. The cvar alone does nothing; a Blueprint-only project can't enable it (needs the `.Target.cs` edit). `[web]` [Epic тАФ Migrate to Iris](https://dev.epicgames.com/documentation/unreal-engine/migrate-to-iris-in-unreal-engine)
- **`DefaultUseSubObjectReplicationList=1` is mandatory** тАФ the engine asserts without it. `[SRC: EngineReplicationBridge.cpp:234]` ┬╖ corroborated `[web]` (Migrate to Iris: subobject list is a hard dependency)
- **Every `AActor` is spatially filtered by DEFAULT; do NOT add `FilterConfigs` to "enable" relevancy** тАФ they *override* the default. Add entries only to make a class always-relevant/unrouted. `[SRC: EngineReplicationBridge.cpp:243]` тЪа **re-verify** (web-unconfirmed; trust the cited line)
- **Spatial separation is plain distance culling** тАФ per-object cull distance (default 150 m 3D); keep zone actors under `MaxNetCullDistance` or they go always-relevant. `[SRC: NetObjectGridFilter.cpp:211,494-500; Actor.cpp:312,101,298]`
- **Live-verified separation at a 2 km offset** (server + 2 clients, Character proxy counts): both at hub = 6; one moved 2 km = 4 (far pawn culled from both other views); both at the offset = 6. `[MEASURED: 2026-07-20]`
- **Iris hard-caps at 128 connections; unchangeable on a Launcher build** тАФ `MaxConnections=128` is a private-header default-arg with no setter/cvar/ini; raising it needs engine source. `[SRC: ReplicationConnections.h:40-45]`
- **Push model is PIE-only on installed builds** (`WITH_PUSH_MODEL=0` in Game/Server targets) тАФ write as if on, don't claim packaged-build push correctness. It's opt-in per property (`DOREPLIFETIME_WITH_PARAMS_FAST` + `MARK_PROPERTY_DIRTY`); miss the mark and it silently reverts to polling. `[SRC: WITH_PUSH_MODEL=0]` ┬╖ `[web]` (StraySpark, community)
- `LevelStreaming` Level Instance cheap replication path: `COND_InitialOnly`. `[SRC: LevelInstanceActor.cpp:87-95]`
- **Level Instance type display-names are SWAPPED:** `Partitioned` shows as **"Embedded"**, `LevelStreaming` shows as **"Standalone"** (`LevelInstanceTypes.h:55-66`) тАФ docs use the UI names, code the identifiers. **`Partitioned` costs zero at runtime** тАФ streaming generation dissolves it and hoists its children into the main grid (`LevelInstanceActor.cpp:222-226`); use it liberally for authoring grouping. Its free-ness gate is **OFPA**, and failure is silent: a non-OFPA source level falls back to level streaming regardless of the dropdown (`LevelInstanceActorDesc.cpp:219-243`) тАФ add a validation check. A **PLA Blueprint** stores **zero** per-instance transform data across N placements (`bInheritPerInstanceData=true`, `PackedLevelActorBuilder.cpp:453-491`). Only `LevelStreaming` gives a real per-instance streaming level тАФ the dynamic-zone / door path. `[SRC: LevelInstanceTypes.h:55-66; LevelInstanceActor.cpp:222-226; LevelInstanceActorDesc.cpp:219-243]`
- **Runtime-spawned actors belong to NO World Partition cell тАФ you spawn it, you destroy it.** WP cell membership is a *cooked* mapping of asset paths (`FWorldPartitionRuntimeCellObjectMapping`) and cell-mutation APIs are `#if WITH_EDITOR` (`WorldPartitionRuntimeCell.h`); **WP will never unload player-built / runtime-spawned content.** Use `ULevelStreamingDynamic::LoadLevelInstance` for a zone *template*, never one level instance per prop. `[SRC: WorldPartitionRuntimeCell.h]`
- **You cannot replicate a placement array** тАФ `net.MaxRepArraySize` caps at 2048 elements / 65535 bytes, and `PerInstanceSMData` carries no `Replicated` specifier so instance arrays never go over the wire on their own (`InstancedStaticMeshComponent.h:168`). Serve large placement sets **out-of-band** (HTTP, keyed by a version hash) and replicate only the hash plus a push-model fast-array of **deltas** on a **dormant** actor (Epic's own topology in `InstancedActors`: `InstancedActorsManager.cpp:217-218` sets `bReplicates=true; SetNetDormancy(DORM_DormantAll)`, deltas at `InstancedActorsData.cpp:1059` тАФ copy the topology, do not adopt the experimental plugin). `[SRC: InstancedStaticMeshComponent.h:168]`
- **Editor `NoRedist` toolsets run Python startup server-side under `-server`**, after the replication system starts тЖТ Iris warns `BroadcastLoadedModulesUpdated() called while there are 1 active ReplicationSystems` (risk of corrupt polymorphic data). Disable editor toolsets for `-server` runs. `[MEASURED: 2026-07-19]`

## 4. Collision
- **ISM collision is per-COMPONENT, not per-instance** тАФ all instances share one collision setting, so there's no runtime per-instance toggle. Set `NoCollision` per component in the PLA source level **before packing**. `[web]` [Epic тАФ Instanced Static Mesh Component](https://dev.epicgames.com/documentation/en-us/unreal-engine/instanced-static-mesh-component-in-unreal-engine)
- **A packed kit creates ISM collision bodies on the dedicated server with no server check** тАФ `OnCreatePhysicsState` calls `CreateAllInstanceBodies()` with no `IsRunningDedicatedServer()` anywhere in the path; rendering is free on a server (`FApp::CanEverRender()` is false under `UE_SERVER`) but collision is not, at one `FBodyInstance` per instance. This is why the above must happen pre-pack. `[SRC: InstancedStaticMesh.cpp:2908-2925]`
- Measured server cost of one packed block: **1,826 ISM components + 665 BodySetups** (BodySetup is per mesh asset, so it understates per-instance cost). `[MEASURED: 2026-07-19]`
- **UE 5.8 collision channels:** ~6 default object + 2 default trace, plus up to **18 CUSTOM** channels тАФ a hard ceiling to budget. (Trial-1 measured 50 total slots including built-ins; the *custom* budget is 18.) `[web]` [Epic тАФ Add a Custom Object Type](https://dev.epicgames.com/documentation/unreal-engine/add-a-custom-object-type-to-your-project-in-unreal-engine) ┬╖ `[MEASURED: 2026-07-19]`
- **Channels are integer indices тАФ inserting/reordering silently reassigns every asset that referenced a shifted index.** Reserve append-only. `[verify]`
- **Collision config (`[/Script/Engine.CollisionProfile]`) is hand-edited in `DefaultEngine.ini` and read at startup** (restart to apply). Opening Project Settings тЦ╕ Collision first rewrites the section from in-memory state and wipes the edit; `ConfigSettingsToolset` can't reach it (custom Slate widget). `[MEASURED: 2026-07-19]` тЪа **re-verify** (rewrite-on-open is project lore)
- **A missing collision profile silently reverts** when set on a component тАФ read-back after setting is the only proof. `[MEASURED: 2026-07-19]`

## 5. Lighting / rendering
- **Sky Atmosphere, Directional Light, Sky Light, height fog, volumetric clouds are scene-global тАФ one active set per world.** Sky Atmosphere supports at most **2 atmospheric directional lights**. `[web]` [Epic тАФ Sky Atmosphere Component](https://dev.epicgames.com/documentation/en-us/unreal-engine/sky-atmosphere-component-in-unreal-engine)
- **A `LevelStreaming` Level Instance streams into the persistent world's ONE `FScene`, so it shares the global day sky** тАФ a Level Instance does NOT grant an independent sky. `[SRC: World.h:1504; SkyAtmosphereRendering.cpp:690-695; RendererScene.cpp:4710-4716; FogRendering.cpp:104]`
- So a night region under a shared day sky is built **spatially**: full enclosure + an **overhead occluder** (an open-top area still sees the global sky and is lit by the global sun) + a **bounded manual-exposure `PostProcessVolume`** + local lights. The occluder must actually seal тАФ if the lid sits above the walls, the day sky leaks the seam and the global sun/skylight still light the interior. `[SRC: architecture.md ┬з7]` ┬╖ `[MEASURED: 2026-07-20]`
- **Supported baked day/night pattern = Lighting Scenario levels** (one visible at a time; each register's lightmaps bake separately). For a Lumen/dynamic day-night, animate the single shared sun/sky instead. `[web]` [Epic тАФ Precomputed Lighting Scenarios](https://dev.epicgames.com/documentation/unreal-engine/using-precomputed-lighting-scenarios-in-unreal-engine)
- **Use MANUAL exposure, locked per register** тАФ auto-exposure re-balances to whatever's in frame (produced an "olive road cast" from a bright-sky-tuned bias). But **locked тЙа crushed**: exposure locked dark to fake night rendered a fully-planted garden (18 trees + 200+ plants) near-black. Stay visible. The "bias" number is empirical, not EV semantics (11 blew white, 4 read night here). `[MEASURED: 2026-07-19 / 2026-07-20]`
- **Packed Level Actors silently drop everything but static meshes and nested level instances** тАФ only two builders exist, ISM and recursive (`PackedLevelActorBuilder.cpp:127-147`); skeletal meshes, **lights**, decals, **Niagara**, audio, splines, volumes and **all Blueprint logic** are dropped with a warning. So door light-spill / neon inside a packed building must be **emissive material**, not a light or Niagara (or a separate external WP light actor). `[SRC: PackedLevelActorBuilder.cpp:127-147]`
- **A Nanite mesh in a PLA becomes a plain ISM, not HISM** (`PackedLevelActorISMBuilder.cpp:70-78`) тАФ correct, since Nanite has its own culling/LOD, though it contradicts Epic docs. Note **Nanite reduces per-draw GPU cost, not draw calls** тАФ instancing is still required. `[SRC: PackedLevelActorISMBuilder.cpp:70-78]`
- **A stale, un-resaved PLA re-runs its construction scripts at load тАФ on the server** (`PackedLevelActor.cpp:65-98`). Grep server logs for the version-mismatch message in CI. `[SRC: PackedLevelActor.cpp:65-98]`
- Baseline is **Lumen + Nanite**. Thin-wall light leak is a Lumen mesh-distance-field resolution problem (thin walls vanish from the SDF). `[verify]`
- **A freshly created empty level (`new_level`) has no sky / sun / skylight, so a SceneCapture FinalColor render comes back fully BLACK.** Add a `DirectionalLight` + `SkyAtmosphere` + `SkyLight` before capturing anything. `[MEASURED: 2026-07-21]` (smoke test)
- **`USceneCaptureComponent2D` does true orthographic capture** тАФ set `projection_type = CameraProjectionMode.ORTHOGRAPHIC` and `ortho_width` (cm), render to a `TextureRenderTarget2D`, export with `RenderingLibrary.export_render_target`. Confirmed genuinely parallel: the image is **pixel-identical across camera distances** (mean diff 0.0 between camera Z 2500 and 4000), whereas perspective changes with distance (diff 1.4тАУ21). Identical cubes at different depths project to identical size. This is the only capture path that gives a non-foreshortened plan/elevation. `[MEASURED: 2026-07-21]`
- **A single directional light cannot light an orthographic *elevation*** тАФ the faces pointing at the camera get no direct light and crush to black at the exposure that suits the sun-lit faces (a full row of cubes rendered pure black while base-colour proved them in-frame). A capture/legibility rig needs a **key + opposing fill directional pair (shadow-casting off) + a skylight**, so every face of an axis-aligned mass catches direct light from one of them. `[MEASURED: 2026-07-21]`
- **A cold / freshly-reposed `SceneCapture2D` renders black on its first `capture_scene()`** тАФ call it several times before exporting (the toolkit primes ~6├Ч). `[MEASURED: 2026-07-21]`

## 6. PIE & dedicated server
- **Net mode is NOT a `StartPIE` parameter** тАФ the engine reads it from the `ULevelEditorPlaySettings` CDO (`/Script/UnrealEd.Default__LevelEditorPlaySettings`): set `playNetMode` / `playNumberOfClients` / `runUnderOneProcess`, then StartPIE. `[SRC: PlayLevel.cpp:991]`
- **The CDO is global editor state тАФ record prior values, restore after the run.** A leak silently changes every later test's net mode. `[verify]`
- **`runUnderOneProcess` inverts:** ON for agent runs (in-process; the watcher subscribes to `PostPIEStarted`); OFF for manual multi-client testing (more representative). `[verify]`
- **Never use Play As Listen Server for authority work** тАФ the host *is* the server, so it can't test "never trust the client". `[web]` [Epic тАФ PIE Multiplayer Options](https://dev.epicgames.com/documentation/unreal-engine/play-in-editor-multiplayer-options-in-unreal-engine)
- **One client can't verify replication** тАФ client 2 catches owner-relevancy / `bOnlyRelevantToOwner` mistakes. A *single* client under a dedicated-server config can misreport `NM_Standalone`; use тЙе1 client with the server explicitly enabled. `[web]` (Epic forum) ┬╖ `[verify]`
- **Player 2 spawns at world origin (0,0,0) with no 2nd PlayerStart** and falls through thin scaled-cube ground тАФ need proper PlayerStarts + solid ground before a real multiplayer walk. `[MEASURED: 2026-07-20]`
- **CQTest** (`PIENetworkComponent.h`) is a networked-PIE harness defaulting to `bIsDedicatedServer=true`, drivable via `AutomationTestToolset`; needs `"CQTest"` in the module `.Build.cs`. `[SRC: PIENetworkComponent.h]`

## 7. Assets (bounds lie, async-load, import)
- **`get_actor_bounds()` on a Packed Level Actor reports the packed asset's ORIGIN, not the rendered geometry** тАФ it read base `-128` for 27 buildings floating 3.25тАУ12.75 m, and three "floating=0" checks passed on a visibly broken level. Measure true base per-ISM: `true_z = t.translation.z + (mb.origin.z тИТ mb.box_extent.z) * t.scale3d.z`. `[MEASURED: 2026-07-19]`
- **When two measurements of one actor disagree, the contradiction IS the finding** тАФ don't pick the convenient one. `[MEASURED: 2026-07-19]`
- **A suspiciously small bound (~2.6 m) is an async-load artifact, not a small asset** тАФ re-query anything implausible. `[MEASURED: 2026-07-19]`
- **A raw obj/gltf in metres imports 100├Ч oversize in cm** (a fountain at 250 m) тАФ place at scale 0.01 if not baked. Free CC BY "scans" arrive raw/oversized/mis-oriented (a bollard 98k tris/11 m; a shutter 4.2M tris/21 m). `[MEASURED: 2026-07-19 / 2026-07-20]`
- **`Texture2D.max_texture_size` caps RUNTIME resolution only** тАФ the full-res source still sits in the `.uasset` (a "2K-capped" texture was 29 MB on disk). Resize the source to shrink the repo. `[MEASURED: 2026-07-19]`
- **Decimating a scan strips UVs тЖТ texture stops mapping.** Enable Nanite on the full mesh instead (preserves UVs, swallows the triangles). `[MEASURED: 2026-07-19]`
- **`AssetTools.duplicate` does NOT rewrite references** тАФ a duplicate still points at the vendor pack. Use Migrate / `migrate_packages` via the Remote Control Python route to curate. `[verify]`

The single most common "my cvar has no effect" cause. `[web]` [Epic тАФ Console Variables in C++](https://dev.epicgames.com/documentation/unreal-engine/console-variables-cplusplus-in-unreal-engine) ┬╖ `[SRC: BaseEngine.ini:2464]`

- The `[SystemSettings]` block in use: `net.Iris.UseIrisReplication=1`, `net.SubObjects.DefaultUseSubObjectReplicationList=1`, `wp.Runtime.EnableServerStreaming=1`, `wp.Runtime.EnableServerStreamingOut=0`. `[verify]`
- A cvar's `ECVF_` flags say when it can be set тАФ **`ECVF_Runtime_ReadOnly` = startup-only.** `EditorAppToolset.SearchCVars` returns the *live* value (proves config actually took). `[verify]`

## 9. Travel between worlds (doors & zone transitions)
- **One `WorldContext` per `GameEngine` in shipping builds тАФ there is no in-process multi-world path.** Running N copies of a world means N processes plus an orchestrator. `[SRC]` (Epic engine comment)
- **Cross-process (server-to-server) travel works, but is never seamless.** `TRAVEL_Absolute` exists for it (`EngineBaseTypes.h:765`, commented *"Traveling from server to server."*); the non-seamless path calls `GEngine->SetClientTravel` (`PlayerController.cpp:5746-5754`). It is a disconnect + reconnect + full map load тАФ a loading screen exactly where the player expected to keep walking. `[SRC: EngineBaseTypes.h:765; PlayerController.cpp:5746-5754]`
- **Seamless travel is same-server + local-package only** тАФ `UWorld::SeamlessTravel`'s map name *"must be relative to the current URL (same server)"* (`World.cpp:9132`) and `FSeamlessTravelHandler::StartTravel` hard-requires `FPackageName::DoesPackageExist(MapName)`, i.e. a local package (`World.cpp:8274-8291`).
