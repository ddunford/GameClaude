# ingest-asset

**Owner: `tech-artist`.** Measurement/audit tool routing (`GeometryAuditTools`, `ContentAuditTools`) and the ingest capability row: `guides/tooling-ue.md`. Hand the placed result to `qa-visual` — never self-approve placement.

Doctrine this enforces: **build ≠ verify** (1) and **complete or descope** (6).

## Procedure

1. **Measure true geometry before trusting anything.** Use the geometry-truth tools (`GeometryAuditTools`) for real bounds/pivot/scale/forward — async-loaded and packed/instanced assets report *fake* actor bounds. Validate scale, pivot, forward axis, and standard skeleton (for skeletal meshes).
2. **Validate against budget and content standards** — polycount vs the asset's budget class, PBR maps present and correct, texture sizes. Record the budget verdict per asset class.
3. **Curate into the project content root** — retarget into `Content/ElseCity/<Category>/` with the naming convention (`SM_`/`SK_`/`M_`/`MI_`/`T_`). Never place straight from a gitignored vendor path. **Use a reference-*fixing* primitive: `EditorAssetLibrary.rename_asset(src, dst)` (MOVE — updates every referencer as it goes), applied to the mesh + its materials + its textures.** ⚠️ **`EditorAssetLibrary.duplicate_asset` / `duplicate_directory` do NOT rewrite cross-asset references** — a duplicated mesh keeps pointing at the *original* `/Game/Fab` materials, and the copy is not self-contained (it renders on this machine and breaks on a fresh clone). Reference fixup happens on rename/move, not on duplicate. Moving trades the local raw cache (re-downloadable via Fab + `fab-acquire`, and gitignored anyway) for a correct self-contained result; delete the emptied vendor pack folders afterward (they hold only orphan redirectors, not a usable re-download source).
4. **Set collision deliberately, per component** where the engine creates bodies server-side with no check — a ninth-floor wall needs none, a ground-floor wall does. Name the cost. For heavy vendor packs, set `NoCollision` in the PLA source level before packing.
5. **Record provenance (mandatory)** — source + licence in the commit message; AI-generated assets keep their prompt.
6. **Decide commit vs declare.** Only content we author or modify moves into `Content/ElseCity/` and is committed. Heavy re-downloadable vendor packs are **declared, not committed** — add to `Tools/required-content.json` (roots, use, sentinel assets) so the startup validator fails loudly on a fresh clone. Never declare a pack that isn't actually placed — it fails the validator forever.

   ⚠️ **Machine-lock trap: a COMMITTED material referencing a declared-not-committed (gitignored) heavy texture renders grey on a fresh clone.** When a committed MI/master leans on a gitignored source texture, that texture must be committed too — but downsized so it fits the LFS budget. **Downsize the exact in-use subset only** (walk the map's transitive deps to find which maps are actually rendered — the whole pack is usually far more than the level uses). The reliable source-shrinking pipeline (`MaxTextureSize`/`lod_bias` do NOT shrink the committed `.uasset` — they only cap the cooked texture, the source bulk stays full-res): **export** (`AssetExportTask` + `unreal.TextureExporterPNG`) → **resize** (PowerShell `System.Drawing`, HighQualityBicubic; worn-surface/noise maps tolerate 1K) → **reimport into the same asset** (`AssetImportTask`, `replace_existing=True`, same `destination_path`/`destination_name` — keeps every referencer intact) → **restore `srgb`/`compression_settings`** (reimport can reset them) → save. Then re-include the specific files in `.gitignore`: git cannot re-include a file under an ignored *directory*, so switch the rule to glob-contents form (`dir/*` + `!dir/specific.uasset` exceptions).
7. **Regenerate the required-content manifest after any placement pass**, or it silently drifts out of date.
8. **Verify references resolve on disk and are self-contained — authoritatively.** ⚠️ **The asset registry's dependency data is STALE for seconds after a bulk duplicate/rename** — a `get_dependencies` call right after the operation returns a false-clean (empty), which reads exactly like "no dangling refs" when refs are in fact still pointing at `/Game/Fab`. Do not trust it. Verify by **inspecting the loaded asset's real object references** (`static_materials[].material_interface.get_path_name()`, and each `MaterialInstanceConstant.texture_parameter_values`), and/or force fresh data with `AssetRegistry.scan_paths_synchronous([...], force_rescan=True)` before a transitive `get_dependencies` walk (hard + soft). A curated asset is self-contained only when its **transitive** `/Game/Fab` dependency count is zero (the MI's parent-material's own functions/parameter-collections count too). Then hand to `qa-visual`.
